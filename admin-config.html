‚öôÔ∏è Permission Configuration

Required Permissions

Live Transcript requires the following Google Workspace permissions:

  • Google Meet Access: To join meetings and access audio
  • Meeting Space Information: To identify speakers and access participant data
  • Audio Stream Access: To capture audio for transcription
  • Third-party Integration: To connect with Deepgram for transcription

OAuth Scopes

The following OAuth scopes are requested by Live Transcript:

https://www.googleapis.com/auth/meetings.conference.media.readonly
https://www.googleapis.com/auth/meetings.conference.media.audio.readonly
https://www.googleapis.com/auth/meetings.conference.media.video.readonly
https://www.googleapis.com/auth/meetings.space.readonly

Data Handling Configuration

  • Audio Processing: Real-time only, no permanent storage
  • Transcription Data: Processed locally, not stored on external servers
  • Participant Information: Used only for speaker identification
  • Meeting Data: Access limited to active meeting sessions

Privacy Controls

Configure privacy settings for your organization:

  • Meeting Consent: Require explicit consent before transcription starts
  • Data Retention: Set policies for any temporary data storage
  • Access Logging: Enable/disable usage logging
  • Third-party Access: Control integration with external services

Permission Errors

Solution: Verify OAuth scopes are approved and user roles have necessary access

Issue: Transcription fails to start
Solution: Check Google Meet API access and third-party service connectivity
